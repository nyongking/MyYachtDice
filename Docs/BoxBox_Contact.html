BoxBox 충돌 — 접촉점 생성 해설

SAT로 충돌을 감지한 뒤, 실제 접촉점(Contact Point)을 어떻게 계산하는지 시각적으로 설명합니다.

전체 흐름

BoxBox 충돌 감지 → 접촉점 생성 전체 파이프라인

SAT 복습 (간단히)

SAT는 "이 축에 투영했을 때 두 박스가 겹치냐?"를 검사합니다. 15개 축 전부 겹치면 충돌입니다.

SAT: 투영 겹침(overlap)이 가장 작은 축이 contact normal이 된다

최소 관통 축의 의미

최소 관통 축(Minimum Penetration Axis)은 "두 박스를 가장 적게 밀어서 분리할 수 있는 방향"입니다. 이 축이 곧 contact.normal이 됩니다.

핵심 아이디어: 이 축이 어디서 왔느냐에 따라 접촉 유형이 결정됩니다.

축 인덱스출처접촉 유형접촉점 개수
0, 1, 2Box A의 면 축면 접촉1~4개
3, 4, 5Box B의 면 축면 접촉1~4개
6 ~ 14외적 (A[i] x B[j])엣지 접촉1개

케이스 분류 — 왜 나누는가?

두 가지 접촉 케이스: 면이 관통하는 경우 vs 엣지끼리 스치는 경우

면 접촉 상세 설명

최소 관통 축이 면 축(인덱스 0~5)일 때의 접촉점 계산입니다.

Step 1. 기준 면(Reference Face) 결정

1 어느 박스의 면이 관통당했는가?

축 인덱스 0~2이면 A의 면이 기준면 (B가 A를 파고듦)
축 인덱스 3~5이면 B의 면이 기준면 (A가 B를 파고듦)

기준면: A의 +x면. normal은 A→B 방향. facePoint는 면 위의 한 점.
// 최소 축이 A의 면 축 (index 0~2)이면
int faceAxisIdx = minAxisIndex;
Vec3 facePoint = posA + minAxisDir * halfA.v[faceAxisIdx];

Step 2. 상대 박스의 8개 꼭짓점 생성

2 파고든 박스(B)의 월드 좌표 꼭짓점 8개

halfExtents의 (+,-) 조합 8가지 → 로컬 좌표 → 회전 + 이동으로 월드 좌표 변환

Box B의 8개 꼭짓점 (halfExtents의 부호 조합)
for (int v = 0; v < 8; ++v)
{
    Vec3 local;
    local.x = (v & 1) ? otherHalf.x : -otherHalf.x;  // 비트 0
    local.y = (v & 2) ? otherHalf.y : -otherHalf.y;  // 비트 1
    local.z = (v & 4) ? otherHalf.z : -otherHalf.z;  // 비트 2

    // 로컬 → 월드
    vertices[v] = otherCenter
        + otherAxes[0] * local.x
        + otherAxes[1] * local.y
        + otherAxes[2] * local.z;
}

Step 3. 각 꼭짓점의 관통 검사

3 기준면 아래로 파고든 꼭짓점만 접촉점

꼭짓점에서 기준면까지의 부호 거리(signed distance)가 음수이면 관통. 관통 깊이 = 부호 거리의 절대값.

기준면 기준으로 안쪽(왼쪽)에 있는 꼭짓점만 접촉점이 된다
for (int v = 0; v < 8; ++v)
{
    Vec3 toVert = vertices[v] - facePoint;
    float penetration = -Vec3Dot(toVert, minAxisDir);

    if (penetration > 0.f)    // 면 아래로 관통
    {
        Contact ct;
        ct.normal = minAxisDir;
        ct.depth  = penetration;
        ct.point  = vertices[v]; // 관통 꼭짓점 자체가 접촉점
        out.push_back(ct);
    }
}

Step 4. 복수 접촉점 — 주사위 예시

접촉 형태에 따라 1~4개의 접촉점이 생성된다. 주사위가 바닥에 놓이면 4개.
왜 복수 접촉점이 중요한가?
물리 시뮬에서 접촉점이 1개이면 박스가 불안정하게 회전합니다 (한 점에서만 힘을 받으니까).
4개이면 안정적으로 놓일 수 있습니다. 주사위가 바닥에 뒹굴다 멈추는 동작에 필수적입니다.

엣지 접촉 상세 설명

최소 관통 축이 외적 축(인덱스 6~14)일 때의 접촉점 계산입니다.

Step 1. 어떤 엣지 쌍인지 결정

1 축 인덱스 → 엣지 쌍 디코딩

축 인덱스 6 + i*3 + j에서 역산:
edgeA = (minAxisIndex - 6) / 3 → A의 i번째 축 방향 엣지
edgeB = (minAxisIndex - 6) % 3 → B의 j번째 축 방향 엣지

박스는 축 방향별로 4개씩, 총 12개의 엣지를 가진다. 외적 축 인덱스로 어떤 엣지 쌍인지 결정.

Step 2. 각 엣지 위의 대표 점 결정

2 같은 축 방향 엣지 4개 중 어떤 것?

예: edgeA = 0 (x축 방향)이면 A에는 x축 방향 엣지가 4개 있다.
이 중 B에 가장 가까운 엣지를 선택한다.
→ 나머지 축(y, z)의 부호를 Dot(axis, d)의 부호로 결정.

d 벡터를 각 축에 투영해서, B 쪽을 향하는 꼭짓점 조합을 선택
// A 엣지의 대표 점
Vec3 ptA = posA;
for (int k = 0; k < 3; ++k)
{
    if (k == edgeA) continue;  // 엣지 방향 축은 건너뜀 (이 축을 따라 이동하니까)
    float sign = (Vec3Dot(axisA[k], d) > 0.f) ? 1.f : -1.f;
    ptA += axisA[k] * (halfA.v[k] * sign);  // B 방향으로 가장 가까운 엣지
}

// B 엣지의 대표 점 (B는 A와 반대 부호)
Vec3 ptB = posB;
for (int k = 0; k < 3; ++k)
{
    if (k == edgeB) continue;
    float sign = (Vec3Dot(axisB[k], d) < 0.f) ? 1.f : -1.f;  // 부호 반대!
    ptB += axisB[k] * (halfB.v[k] * sign);
}

Step 3. 두 직선의 최근접점 (Closest Point on Two Lines)

3 3D 공간에서 두 직선 사이의 가장 가까운 점

두 엣지를 무한 직선으로 취급하고, 두 직선 위에서 거리가 최소가 되는 점 쌍(closestA, closestB)을 구한 뒤, 그 중점을 접촉점으로 사용합니다.

두 엣지의 최근접점 쌍을 구하고, 그 중점이 접촉점

두 직선의 최근접점은 연립방정식으로 구합니다:

두 직선:
LineA(t) = ptA + dirA * tA
LineB(s) = ptB + dirB * tB

w = ptA - ptB 로 놓으면:
a = Dot(dirA, dirA) — 항상 1 (정규화된 축)
b = Dot(dirA, dirB)
c = Dot(dirB, dirB) — 항상 1
d = Dot(dirA, w)
e = Dot(dirB, w)

denom = a*c - b*b
tA = (b*e - c*d) / denom
tB = (a*e - b*d) / denom

tA, tB를 각 halfExtent 범위로 클램프한 뒤 최종 점 계산.
