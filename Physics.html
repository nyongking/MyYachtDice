Symplectic Euler 적분

Contact 구조체

struct Contact { Vec3 point; // 접촉점 (월드 좌표) Vec3 normal; // A→B 방향 충돌 법선 float depth; // 관통 깊이 (양수) RigidBody* bodyA; RigidBody* bodyB; };
Phase 4 충돌 응답 (Sequential Impulse)

충격량 계산

// 접촉점까지의 벡터 rA = contact.point - bodyA.position rB = contact.point - bodyB.position // 상대 속도 relVel = (vB + wB×rB) - (vA + wA×rA) sepVel = Dot(relVel, normal) // 분리 중이면 처리 불필요 if (sepVel > 0) return; // 충격량 크기 j = -(1 + e) × sepVel ÷ ( invMassA + invMassB + Dot(n, (invIA × (rA×n)) × rA) + Dot(n, (invIB × (rB×n)) × rB) ) // 선속도 갱신 vA -= j × normal × invMassA vB += j × normal × invMassB // 각속도 갱신 wA -= invIA × (rA × (j × normal)) wB += invIB × (rB × (j × normal))

마찰 (Coulomb 모델)

// 접선 방향 (마찰 방향) tangent = relVel - Dot(relVel, normal) × normal tangent.Normalize() // 마찰 충격량 크기 (j 재사용) jt = -Dot(relVel, tangent) ÷ (invMassA + invMassB + ...) // Coulomb 마찰 한계 클램프 μ = sqrt(frictionA × frictionB) jt = Clamp(jt, -μ×j, μ×j) // 적용 vA -= jt × tangent × invMassA vB += jt × tangent × invMassB

Penetration 보정

// slop보다 깊이 파고든 경우만 보정 slop = 0.01f beta = 0.2f // 보정 비율 correction = max(depth - slop, 0) ÷ (invMassA + invMassB) × beta × normal posA -= invMassA × correction posB += invMassB × correction
반복 횟수: Sequential Impulse는 모든 Contact에 대해 10~20회 반복(Gauss-Seidel). 반복할수록 정확도 향상, 실시간 안정성 확보.
Phase 5 시스템 통합

PhysicsWorld::Step(dt) 루프

Step(dt): 1. 외력 적용 (중력 등) 2. Broad Phase → 후보 쌍 목록 3. Narrow Phase → Contact 목록 4. Constraint Solver (N회 반복) └─ 각 Contact: 충격량 계산 + 적용 5. 적분 (모든 RigidBody) 6. ClearAccumulators()
스레드 정책: Step()은 thread-safe를 보장하지 않는다. 호출자가 단일 스레드에서 호출하거나, 외부에서 mutex로 보호해야 한다.

설계 원칙: PhysicsLib은 순수 물리만 담당한다. Bias 로직은 호출자(GameEngineLib 또는 Client)에서 Step() 전후로 주입하는 형태로 구현한다.

호출 구조

매 프레임: DiceController::InjectBias(rigidBody, targetFace) ← 외부에서 조작 주입 PhysicsWorld::Step(dt) ← 순수 물리 계산

Phase A  속도 빠를 때

정상 물리 그대로. 개입 없음.

조건동작
angularVelocity.Length() > highThresholdBias 비활성

Phase B  속도 줄어들 때

목표 면 근처에서 감쇠를 높여 자연스럽게 브레이크.

조건동작
angularVelocity.Length() < highThresholdangularDamping 증가
목표 면이 위를 향할수록 damping을 더 강하게 적용 → 원하는 방향 근처에서 자연스럽게 속도 감소

Phase C  거의 멈출 때

약한 보정 토크를 주입해 목표 방향으로 미세 유도.

조건동작
angularVelocity.Length() < lowThreshold보정 토크 ApplyTorque()
현재 face normal과 목표 face normal의 각도 차이로 토크 방향 계산. 세기는 작게 유지해야 자연스러워 보임.

Phase D  완전 정지 후

오차 보정용 orientation 스냅. 아주 작은 각도만 허용.

조건동작
angularVelocity ≈ 0 && linearVelocity ≈ 0목표 Quat으로 Slerp 보정
주의: 각도 차이가 클 경우 티가 날 수 있음. Phase B·C에서 충분히 유도한 뒤 미세 보정 용도로만 사용.

전체 흐름

angularVelocity.Length(): > highThreshold → [A] 정상 물리, 개입 없음 > lowThreshold → [B] angularDamping 증가 목표 면 방향일수록 damping 강하게 > 0 (거의 정지) → [C] 약한 보정 토크 주입 현재 face → 목표 face 방향으로 유도 == 0 (완전 정지) → [D] Slerp로 미세 오차 보정 각도 차이 작을 때만 적용
